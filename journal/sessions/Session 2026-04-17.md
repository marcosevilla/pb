- **Task list redesign:** flattened subtasks (no indent) with inline `CornerDownRight` icon for subtask rows; parent rows get `ListTree` + `done/total` badge. Same width for every row. Applied across TasksPage, InboxPage, TodayPage Tasks section.
- **Hover affordance migration:** per-row action buttons (focus/edit/add-subtask/delete) moved entirely into the bulk action bar. When 1 task selected → full action set. When 2+ selected → focus-queue (extended focusStore with queue field so next task auto-starts after current completes), status, move, delete. Grip + checkbox hover-only, content flush-left.
- **Priority bars (Linear-style):** replaced colored dots with 3-bar cell-signal indicator, color-agnostic (foreground/muted). Applied everywhere priorities show — row, picker, dialogs.
- **Todoist migration:** schema v15 added `external_id`/`external_source` columns, Rust migration module (`todoist_migration.rs`) does preview + upsert of projects/sections/tasks/subtasks with label concatenation + recurring note preservation. Settings UI with preview card, option toggles, AlertDialog confirmation, post-import summary. Filter hides migrated tasks from the live Todoist panel. Also fixed a pagination bug: cursor URL-encoding was broken (`+`/`/` in base64 cursors mangled by naive string concat), hitting an exactly-500 truncation. Rewrote with `reqwest::Url::query_pairs_mut()` + explicit `limit=200`.
- **Motion system:** design critique via `/interface-craft` surfaced 5 opportunities. Shipped all five: unified motion tokens (`--transition-fast/base/slow`, `--ease-entrance`, `--ease-exit`) referenced by every keyframe; wired `task-complete-exit` to actual completion (StatusDropdown + BulkActionBar delay mutation behind 580ms animation, 40ms stagger on batches); drag-reorder neighbors now use the entrance curve; micro-pulse on status + priority change via `animate-count-pulse`; staggered row entrance on lists (`animate-row-enter`, 25ms per row capped at 15).
- **Other scattered fixes:** horizontal overscroll bounce killed with `overscroll-behavior: none` + `overflow: hidden` on `html, body`; no-horizontal-scroll guarantee via `min-w-0` on `<main>` + `overflow-x-hidden` on scroll container; unified checkbox padding between notes + tasks; checkbox-absolute-overlay causing overlap with leading content swapped back to in-flow with opacity-toggle.
- **Key insight:** the ROI on unifying system tokens is paid back by the NEXT change. The motion tokens took 20 min to wire; the next time we want to change easing across the app it's a 1-line edit. Same with PageHeader — one visual change now propagates to 9 pages. The cost of the abstraction is always front-loaded, the benefit is compound.
